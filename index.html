A year with angular

Chris Wolfe

June 20, 2015

The project

Medium sized CRUD application centered around financial reporting.

Started as team of 2, balloned to more than 10

The good

The framework is explicit

Dependency injection and testing work well together

The library has matured and there are good resources to learn it

Much thought has gone into simplifying the writing of data driven applications

Templates are just simple html with curly braces, not a new language

There are semi clear distinctions between components

  • Services hold state and logic that needs be shared
  • Controllers push the data contained in services to templates
  • Directives manipulate the DOM

The bad

There is always more than one way to do things

Error messages are unclear

Semi-frequent, large, breaking API changes

In my experience, it has been difficult for newcomers to learn

Many tutorials available tell users what to do, without teaching about the concepts. This leads to cargo-culting instead of understanding.

Tips when using angular

Services should contain almost all logic. Use them to share information between things

If you are writing directives, try to use isolate scope whenever possible

Do not rely on inheritance, that is, avoid needing to know about parents.

Should I start a brand new long-term angular project now?

No, I would not start with an Angular 1.4 for a new medium to large project.

But!

It is a manageable well thought out library that has nice abstractions. If you are planning something shorter term that won't need a lot of long term maintenence, it can be a good choice
