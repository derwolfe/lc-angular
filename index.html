A year with angular

Chris Wolfe

June 20, 2015

The project

Medium sized CRUD application centered around reporting.

Started as team of 2, balloned to 8

The good

The framework does not rely on "magic"

Dependency injection is explicit

Most components are easy to test

The library has been around long enough that there are good resources to learn it

It is a domain specific language for writing web applications

Templates are html with curly braces

There are semi clear distinctions between components

  • Services hold state and logic that needs be shared
  • Controllers push the data contained in services to templates
  • Directives manipulate the dom

The bad

There is always more than one way to do things

Error messages are unclear

Breaking API changes

It has been difficult for newcomers to learn

Tips for using angular

Services should contain almost all logic. Use them to share information between things

If you are writing directives, try to use isolate scope whenever possible

Do not rely on inheritance, that is, avoid needing to know about parents.

Should I start a new angular project now?

No, I would not start with angular 1.4 for a new project
